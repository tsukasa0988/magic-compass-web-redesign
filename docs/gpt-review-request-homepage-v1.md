## 你要幫忙審的重點

請你對照「原始參考稿」與「目前 Codex 實作版」，逐項檢查以下內容：

### 1. Hero 首屏
- 是否仍保有 CFD 平台主視覺感
- 左右視覺重心是否合理
- 主標題字級、行高、寬度是否太重或太擠
- 裝置圖比例、位置、壓迫感是否需要調整
- CTA 層級是否足夠明確
- 首屏是否過於偏插畫感，沒有貼近原稿的成熟交易平台感

### 2. 核心數據條
- 高度、留白、字級比例是否接近參考圖
- 圖示與數值是否過於粗重
- 是否需要更像正式金融平台的資訊條，而不是功能卡

### 3. 熱門市場報價區
- 卡片尺寸、密度、間距是否貼近參考圖
- tab 樣式是否正確
- 整體是否保留市場感但不過度工具化

### 4. 平台介紹 / 核心賣點區
- 左文右圖比例是否合理
- 優勢卡片的節奏是否夠乾淨
- 是否需要更接近原稿中的模組層次

### 5. 平台下載區
- 卡片排版是否太鬆或太空
- icon、標題、按鈕的層級是否需要更明確
- 是否接近原稿「正式平台下載入口」而不是功能展示卡

### 6. 即時報價表
- 表格高度、欄距、字級是否太鬆
- 視覺上是否還能再更精煉
- 是否保留了「簡版行情表」的定位

### 7. 品牌視頻 + 開戶流程
- 雙欄比例是否合理
- 視頻封面是否需要更接近原稿的品牌感
- 開戶流程是否太像後台表單模組，而非品牌網站區塊

### 8. 信任背書區
- 卡片表現是否太普通
- 是否足夠傳達信任背書，但又沒有做成過長說明

### 9. 底部 CTA + Footer
- CTA 是否足夠收尾
- Footer 的資訊結構是否正式
- 是否需要更接近金融品牌網站，而不是一般 SaaS 頁面

### 10. WAP 響應式表現
- 首屏在手機上的視覺節奏是否正確
- Hero 主標是否過大
- 按鈕是否過寬或過擠
- 模組縱向節奏是否順
- 哪些區塊在手機版需要重新排序、縮高或改密度

## 你輸出的格式

請不要只給抽象評論，請直接輸出「Codex 可執行修正清單」。

格式請嚴格使用以下結構：

### A. 保留不動
列出目前這版哪些地方可以先保留，不需要改。

### B. 必改項目
請列出本輪最需要優先修正的項目，依重要度排序。

每一項都請包含：
- 區塊名稱
- 問題描述
- 具體修改方向
- 修改後應更接近什麼感覺

### C. 可優化但非本輪必改
列出可以下一輪再處理的細節。

### D. 給 Codex 的直接施工指令
請最後整理成一份「可以直接貼給 Codex」的執行指令。
