目前 repo 狀態偏「企劃/規格階段」，尚未進入前端程式開發階段。

## 2. 目前不建議直接進入最終切版的原因

現有文件已足夠讓我開始做結構與開發前置，但還不足以直接做最終版 UI，原因是仍缺少以下定稿資訊：

- PC 首頁完整視覺稿
- WAP 首頁完整視覺稿
- 字體方案
- 精準色票與按鈕狀態
- 區塊間距、卡片樣式、圓角與陰影規格
- Hero 視覺素材風格與構圖基準

如果現在直接進入高保真切版，後續視覺稿一出，很高機率需要重做版型、字級、留白、圖片比例與 CTA 層級。

## 3. 建議你現在交給 GPT 的產出

建議 GPT 下一輪先輸出這 2 份主稿：

1. `PC 首頁長版視覺定稿圖`
2. `WAP 首頁長版視覺定稿圖`

這兩份稿件請不要只做概念海報，而要做成「可落地切版」的設計提案稿。

## 4. GPT 視覺稿必須包含的內容

### A. 共用視覺原則

- 網站定位：品牌型、強信任感的 CFD 平台官網首頁
- 視覺風格：國際感、專業、穩定、金融科技感、高級但不浮誇
- 首屏方向：圖二做骨架，圖一補首屏交易感
- 主色：深藍 / 藍黑
- 輔色：亮藍少量點綴
- CTA 強調色：金色
- 背景策略：白底與深色區塊混搭
- 避免：過重工具感、老派經紀商模板感、勵志旅程型主視覺、過多 ticker 雜訊

### B. 首頁模組順序

請 GPT 依照以下固定順序出稿：

1. Header
2. Hero 主 Banner
3. 核心數據條
4. 熱門市場報價 / 市場焦點
5. 平台介紹 / 核心賣點
6. 核心優勢區
7. 平台下載 / 終端支持
8. 即時報價表（簡版）
9. 品牌視頻 + 開戶流程
10. 信任背書 / 品牌實力
11. 底部 CTA
12. Footer

### C. 每個模組至少要回答的內容

#### 1. Header
- Logo 放置方式
- 主導航樣式
- 語言切換位置
- `登入` 與 `立即開戶` 的層級
- 手機版 header 展開方式

#### 2. Hero 主 Banner
- 主標、副標、雙 CTA 的版位
- 是否輪播，以及第一張主 Banner 的主構圖
- 交易終端、行情圖、全球市場元素如何入畫
- PC 與 WAP 的構圖差異

#### 3. 核心數據條
- 顯示 4 個關鍵數據或 4 個信任指標
- 卡片形式或橫條形式
- 深色區/淺色區中的呈現方式

#### 4. 熱門市場報價 / 市場焦點
- Tab 樣式
- 商品卡片樣式
- 小型走勢圖是否存在
- 手機版如何垂直堆疊

#### 5. 平台介紹 / 核心賣點
- 文字與裝置圖的比例
- 是否做左右交錯版型
- 輕量 CTA 的樣式

#### 6. 核心優勢區
- 4 格內卡片的排法
- icon 風格
- 標題與描述的資訊密度

#### 7. 平台下載 / 終端支持
- Android / iOS / Desktop / MT4 的呈現形式
- 下載按鈕層級
- QR code 是否需要預留位置

#### 8. 即時報價表（簡版）
- 表格精簡程度
- tab 切換視覺
- 與整站品牌感是否一致

#### 9. 品牌視頻 + 開戶流程
- 雙欄還是上下堆疊
- 視頻封面樣式
- 3 步開戶流程的視覺節奏

#### 10. 信任背書 / 品牌實力
- 監管資訊或品牌徽章的呈現方式
- 是否以 badge/card/stat 模組處理
- 避免做成過重長文說明

#### 11. 底部 CTA
- 一句收尾口號
- `立即開戶` / `下載平台` 的按鈕組合
- 背景氛圍與頁尾收口方式

#### 12. Footer
- 導覽欄位分組
- 社群連結區
- 法律、風險、隱私資訊的資訊密度
- 手機版簡化方式

## 5. GPT 額外要一併給出的規格

除了視覺圖本身，請 GPT 一併補出以下內容，這些是我切版時最需要的：

- 字體建議：英文標題、英文內文、中文替代方案
- 色票：主色、輔色、背景色、文字色、邊框色、CTA 色
- 按鈕規格：主按鈕、次按鈕、hover、disabled
- 卡片規格：圓角、陰影、邊框、內距
- 區塊規格：section 間距、容器寬度、PC/WAP 斷點建議
- 圖像方向：Hero 主要視覺素材風格說明
- 文案規格：每個 section 的標題長度、副標長度、按鈕文案

## 6. Codex 下一步的最佳施工順序
