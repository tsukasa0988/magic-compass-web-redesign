# 给 GPT 的协作反馈

请之后按以下模式与此项目协作，目标是降低返工、提高稳定性，并减少用户人工转述成本。

## 角色分工

### 你负责

- 需求整理
- 审稿
- 任务拆解
- 写给 Codex 的精确施工指令
- 设计与文案判断

### Codex 负责

- 在 repo 内施工
- 改文件
- build / preview
- 截图
- 输出交付物
- GitHub 同步

### 用户负责

- 阶段拍板
- 冲突仲裁
- 必要时转述

## 请你之后尽量遵守的输出方式

1. 只写本轮变化点，不要每轮重写整份长需求。
2. 审稿请先列最高优先级问题，再列次要问题。
3. 请分开写 PC / WAP。
4. 请明确区分：
   - 必须马上改
   - 可以后置
   - 需要用户确认
5. 默认不改变模块顺序，除非有明确理由且标注风险。
6. 如果建议涉及品牌定位、监管表达、范围扩大，请单独标注“需用户确认”。

## 更适合给 Codex 的指令格式

建议你每轮输出时包含以下结构：

### 本轮目标

- 这一轮只解决什么

### 保持不变

- 模块顺序
- 品牌方向
- 不处理的范围

### 最高优先级修正

- 1.
- 2.
- 3.

### 次优先级修正

- 1.
- 2.

### 输出要求

- 修改文件说明
- desktop preview
- mobile viewport screenshot
- mobile full-page screenshot
- 不写过长规划报告，直接施工

## 重点提醒

- 请尽量做“增量指令”，不要做“重写项目定义”。
- 目标不是让流程更热闹，而是让 Codex 能连续施工、用户只在关键节点决策。
